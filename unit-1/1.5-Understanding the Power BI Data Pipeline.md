# Introduction

A **Data Pipeline** is the sequence of steps through which raw data is collected, processed, transformed, analyzed, and finally presented as reports and dashboards.

In Power BI, the data pipeline ensures that data flows smoothly from multiple sources into meaningful visualizations that support business decision-making.

Think of the data pipeline like a **water pipeline**:

* **Water Source** → Raw Data
* **Water Treatment Plant** → Data Cleaning & Transformation
* **Water Storage Tank** → Data Model
* **Water Distribution** → Reports & Dashboards
* **Consumers** → Business Users

Similarly, Power BI collects raw data, processes it, and delivers useful insights to decision-makers.

# 1. Data Sources and Connectivity

## What are Data Sources?

A **data source** is any location where data is stored.

Power BI can connect to a wide variety of data sources, making it suitable for organizations of all sizes.

### Types of Data Sources

### 1. Files

Examples:

* Excel (.xlsx)
* CSV
* XML
* JSON
* PDF
* Text files

**Example**

A company stores monthly sales data in an Excel workbook. Power BI imports this data to create sales reports.

---

### 2. Databases

Examples:

* SQL Server
* Oracle
* MySQL
* PostgreSQL
* IBM DB2

**Example**

An online shopping company stores order details in SQL Server. Power BI connects directly to the database for analysis.

---

### 3. Cloud Services

Examples:

* Azure SQL Database
* SharePoint Online
* OneDrive
* Google Analytics
* Salesforce
* Dynamics 365

**Example**

A marketing team analyzes website traffic by connecting Power BI to Google Analytics.

---

### 4. Web Services and APIs

Power BI can retrieve live data from websites and APIs.

Examples:

* Weather API
* Stock Market API
* Twitter API

**Example**

A logistics company displays live weather information to plan deliveries.

---

### 5. Enterprise Applications

Examples:

* SAP
* Oracle ERP
* Microsoft Dynamics
* Tally (through connectors)

**Example**

A manufacturing company imports inventory and production data from SAP.

# Connectivity Modes in Power BI

Power BI supports three main connectivity modes.

## A. Import Mode

Data is copied into Power BI.

### Advantages

* Fast performance
* Offline analysis
* Full Power BI features

### Disadvantages

* Data becomes outdated unless refreshed.

**Example**

A company imports monthly sales data from Excel every month.

---

## B. DirectQuery Mode

Power BI does not store the data.

Instead, every visual sends a query to the database in real time.

### Advantages

* Always shows current data.
* Suitable for large databases.

### Disadvantages

* Depends on database speed.

**Example**

A bank displays live ATM transaction reports.

---

## C. Live Connection

Power BI connects directly to Analysis Services or existing Power BI datasets.

### Advantages

* Single version of truth
* Centralized management

### Example

Multiple departments use the same centralized financial model.

---

# Comparison of Connectivity Modes

| Feature                 | Import                | DirectQuery         | Live Connection |
| ----------------------- | --------------------- | ------------------- | --------------- |
| Stores Data in Power BI | Yes                   | No                  | No              |
| Performance             | Fast                  | Depends on database | Fast            |
| Real-Time Data          | No                    | Yes                 | Yes             |
| Offline Access          | Yes                   | No                  | No              |
| Best Use                | Small/Medium datasets | Large databases     | Enterprise BI   |

# 2. Data Acquisition Process

## What is Data Acquisition?

Data acquisition is the process of collecting data from one or more sources before analysis.

It is the first stage in the Power BI data pipeline.

### Steps in Data Acquisition

### Step 1: Identify Data Sources

Determine where the required data is stored.

Example:

* Sales → SQL Server
* Customers → Excel
* Products → SharePoint

---

### Step 2: Connect to the Source

Power BI provides built-in connectors.

Examples:

* Excel
* SQL Server
* Web
* Oracle

---

### Step 3: Authenticate

Power BI verifies user access using:

* Username and password
* Windows Authentication
* OAuth
* API Keys

---

### Step 4: Select Required Data

Choose:

* Tables
* Sheets
* Views
* Queries

Example:

Instead of importing the entire database, only the **Sales** and **Customers** tables are selected.

---

### Step 5: Load or Transform

Users can:

* Load data directly
* Open Power Query to clean and transform data before loading

---

# Example

A supermarket stores:

* Sales in SQL Server
* Employee data in Excel
* Customer feedback in SharePoint

Power BI connects to all three sources and combines them into one report.

# 3. ETL Concepts in Power BI

## What is ETL?

**ETL** stands for:

* **Extract**
* **Transform**
* **Load**

It is the process of preparing data for analysis.

Power BI performs ETL mainly through **Power Query**.

---

## Step 1: Extract

Data is collected from multiple sources.

Example:

A company imports:

* Sales from SQL Server
* Customer data from Excel
* Product information from SAP

---

## Step 2: Transform

Data is cleaned and prepared.

Common transformations include:

* Remove duplicates
* Rename columns
* Merge tables
* Split columns
* Filter unwanted records
* Change data types
* Replace missing values

### Example

Original Data

| Name  | Sales |
| ----- | ----- |
| Ravi  | 25000 |
| Ravi  | 25000 |
| Kumar | Null  |

After Transformation

| Name  | Sales |
| ----- | ----- |
| Ravi  | 25000 |
| Kumar | 0     |

---

## Step 3: Load

The cleaned data is loaded into the Power BI Data Model.

After loading, users can:

* Build relationships
* Create DAX measures
* Design reports

---

# Benefits of ETL

* Improves data quality
* Removes inconsistencies
* Increases reporting accuracy
* Reduces duplicate records
* Saves analysis time

# 4. End-to-End Reporting Workflow

The reporting workflow shows how data moves from its source to the final dashboard.

## Step 1: Data Collection

Collect data from:

* Excel
* SQL Server
* Web APIs
* ERP Systems

**Example:** A retail company collects sales, inventory, and customer data.

---

## Step 2: Data Import

Connect Power BI to the required data sources.

---

## Step 3: Data Transformation

Use Power Query to:

* Clean data
* Remove duplicates
* Handle missing values
* Merge tables
* Rename columns

---

## Step 4: Data Modeling

Create relationships between tables and define business calculations using DAX.

**Example:**

Customers → Orders → Products → Sales

---

## Step 5: Report Development

Create visualizations such as:

* Bar charts
* Line charts
* Pie charts
* Maps
* KPI Cards
* Tables

---

## Step 6: Dashboard Creation

Pin important report visuals to a dashboard.

**Example Dashboard**

* Total Revenue
* Total Profit
* Monthly Sales Trend
* Top Products
* Regional Performance

---

## Step 7: Publish to Power BI Service

Publish reports to the cloud for secure sharing.

Users can:

* View reports online
* Collaborate with colleagues
* Schedule automatic refresh
* Control access

---

## Step 8: Access on Mobile

Managers can view reports using the Power BI Mobile app.
