# Introduction

After importing and cleaning data, the next step is **Data Transformation**. Data transformation involves converting raw data into a structured and meaningful format suitable for analysis.

Power BI uses **Power Query Editor** to perform various transformation operations without changing the original data source.

### Definition

**Data Transformation** is the process of modifying, restructuring, or converting data into a format that is suitable for analysis and reporting.

---

# Why is Data Transformation Important?

Raw data is often:

* Combined into a single column
* Stored in incorrect formats
* Missing derived information
* Difficult to analyze

Data transformation helps:

* Improve readability
* Simplify analysis
* Increase data consistency
* Support business calculations
* Build better dashboards

# 1. Splitting and Merging Columns

## A. Splitting Columns

### Definition

**Splitting a column** means dividing one column into two or more separate columns based on a delimiter, number of characters, or position.

### Why Split Columns?

Many datasets store multiple values in one column.

Example

| Full Name    |
| ------------ |
| Ravi Kumar   |
| Anita Sharma |

Instead of one column, we may need:

| First Name | Last Name |
| ---------- | --------- |
| Ravi       | Kumar     |
| Anita      | Sharma    |

---

## Methods of Splitting

### 1. Split by Delimiter

Delimiter examples:

* Space
* Comma
* Hyphen
* Slash

### Example

Original Data

| Full Name  |
| ---------- |
| Ravi Kumar |

Split by Space

| First Name | Last Name |
| ---------- | --------- |
| Ravi       | Kumar     |

---

### 2. Split by Number of Characters

Example

Employee Code

EMP1001

Split after 3 characters

| Prefix | Number |
| ------ | ------ |
| EMP    | 1001   |

---

### 3. Split by Position

Example

Product Code

AB123456

Split after the second character.

---

## Steps in Power Query

1. Select the column.
2. Transform → Split Column.
3. Choose:

   * By Delimiter
   * By Number of Characters
   * By Positions
4. Apply the transformation.

---

# B. Merging Columns

### Definition

Merging combines two or more columns into a single column.

---

### Example

Original Data

| First Name | Last Name |
| ---------- | --------- |
| Ravi       | Kumar     |

After Merge

| Full Name  |
| ---------- |
| Ravi Kumar |

---

### Steps

1. Select multiple columns.
2. Transform → Merge Columns.
3. Choose separator:

   * Space
   * Comma
   * Hyphen
4. Create merged column.

---

### Business Example

A company stores:

| City | State |

To display addresses, combine them into:

**Bangalore, Karnataka**

---

# Advantages of Splitting and Merging

* Better data organization
* Easier filtering
* Improved reporting
* Supports accurate analysis

# 2. Data Type Transformations

## What is a Data Type?

A **data type** defines the kind of value stored in a column.

Power BI automatically detects data types but sometimes they must be corrected.

---

## Common Data Types

| Data Type      | Example             |
| -------------- | ------------------- |
| Whole Number   | 100                 |
| Decimal Number | 25.75               |
| Currency       | ₹1,250.50           |
| Date           | 10/07/2026          |
| Date & Time    | 10/07/2026 10:30 AM |
| Text           | Bangalore           |
| True/False     | Yes/No              |

---

## Why Change Data Types?

Incorrect data types can:

* Produce calculation errors
* Prevent chart creation
* Affect sorting and filtering

---

### Example

Before

| Sales   |
| ------- |
| "25000" |

Stored as Text

Cannot calculate Total Sales.

---

After Changing to Number

| Sales |
| ----- |
| 25000 |

Now calculations work correctly.

---

## Changing Data Types

Steps

1. Select column.
2. Transform → Data Type.
3. Choose:

   * Whole Number
   * Decimal Number
   * Currency
   * Date
   * Text

---

### Business Example

A banking system imports transaction amounts as text.

Changing them to Currency enables total balance calculations.

---

# Advantages

* Accurate calculations
* Better filtering
* Proper sorting
* Improved chart creation

# 3. Creating Custom Columns

## Definition

A **Custom Column** is a new column created using formulas based on existing columns.

Power Query uses the **M Language** to create custom columns.

---

## Why Use Custom Columns?

Business users often need additional information that is not present in the original dataset.

---

### Example

Original Data

| Quantity | Price |
| -------- | ----- |
| 5        | 200   |

Custom Column

Total Amount

Formula

Quantity × Price

Result

| Quantity | Price | Total Amount |
| -------- | ----- | ------------ |
| 5        | 200   | 1000         |

---

## Creating a Custom Column

Steps

1. Add Column
2. Custom Column
3. Enter formula
4. Click OK

---

### Example Formula

```text
=[Quantity] * [Price]
```

---

### Example 2

Employee Age

Current Year − Birth Year

Result

Age Column

---

### Example 3

Profit

Revenue − Cost

---

## Business Example

An e-commerce company stores:

| Cost | Selling Price |

A custom column calculates:

Profit = Selling Price − Cost

---

# Advantages

* Adds derived information
* Eliminates manual calculations
* Supports business analysis

# 4. Conditional Columns

## Definition

A **Conditional Column** creates values based on conditions, similar to IF-ELSE statements in programming.

---

### Example

If Sales > ₹50,000

Status = High Sales

Else

Status = Low Sales

---

Original Data

| Sales |
| ----- |
| 65000 |
| 25000 |

After Conditional Column

| Sales | Category   |
| ----- | ---------- |
| 65000 | High Sales |
| 25000 | Low Sales  |

---

## Steps

1. Add Column
2. Conditional Column
3. Define conditions

Example

IF

Sales > 50000

THEN

High

ELSE

Low

---

## Multiple Conditions

Example

IF Sales >100000

Excellent

ELSE IF Sales >50000

Good

ELSE

Average

---

### Business Example

A company classifies customers.

| Purchase Value    | Category |
| ----------------- | -------- |
| Above ₹1,00,000   | Premium  |
| ₹50,000–₹1,00,000 | Gold     |
| Below ₹50,000     | Silver   |

Conditional Columns automate this classification.

---

# Calculated Fields

## What are Calculated Fields?

Calculated fields are values created using formulas after data is loaded into the Power BI Data Model.

They are created using **DAX (Data Analysis Expressions)**.

Unlike Custom Columns (created in Power Query), calculated fields are created in Power BI after the data is loaded.

---

### Example

Sales Table

| Revenue | Cost  |
| ------- | ----- |
| 100000  | 75000 |

Calculated Field

Profit = Revenue − Cost

---

### Difference Between Custom Columns and Calculated Fields

| Feature     | Custom Column                  | Calculated Field                |
| ----------- | ------------------------------ | ------------------------------- |
| Created In  | Power Query                    | Power BI Data Model             |
| Language    | M Language                     | DAX                             |
| Calculated  | During data load               | After data is loaded            |
| Usage       | Data preparation               | Business analysis               |
| Performance | Faster for static calculations | Better for dynamic calculations |

# Real-World Case Study

## ABC Electronics

### Problem

Sales data contained:

* Full customer names in one column
* Prices stored as text
* No profit calculation
* No customer classification

### Solution

Using Power Query:

* Split Full Name into First Name and Last Name.
* Changed Price from Text to Currency.

* Created a DAX calculated field for Profit Margin.
* Built interactive sales dashboards.

* Improved reporting accuracy.
* Faster customer segmentation.
* Better business insights.

## Classroom Practical (2 Hours)

Students will work with a sample **Sales Dataset** and perform the following tasks:

1. Import the dataset into **Power Query**.
2. Split the **Full Name** column into **First Name** and **Last Name**.
3. Merge **City** and **State** into a single **Location** column.
4. Convert the **Sales Amount** column from **Text** to **Currency**.
5. Create a **Custom Column** to calculate **Total Amount = Quantity × Unit Price**.
6. Create a **Conditional Column** to categorize sales as **High**, **Medium**, or **Low** based on sales value.
7. Load the transformed data into Power BI and verify the results through a simple report.

This practical session reinforces the concepts of data transformation and prepares students for real-world business intelligence projects.
